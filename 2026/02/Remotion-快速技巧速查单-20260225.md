### 技巧 2: 超流畅的文本动画

**问题**：文字出现不流畅，导致跳帧

```typescript
// ❌ 错误做法（导致重新渲染）

    <CharacterComponent key={i} char={char} delay={i * 0.05} />
  ));

const CharacterComponent = memo(({ char, index }) => {

}, (prev, next) => prev.char === next.char && prev.index === next.index);
```

**关键点**：使用 `memo` 避免每一帧都重新渲染所有字符。

### 技巧 5: 文本自适应容器宽度

**问题**：长文本超出容器，布局混乱

**解决方案**：

```typescript
import { useTextSize } from '@remotion/media-utils';

export const ResponsiveText = ({ text, maxWidth = 1200 }) => {
  const { width: textWidth } = useTextSize({
    text,
    fontSize: 48,
    fontFamily: 'Arial'
  });

  // 自动调整字号
  let finalFontSize = 48;
  if (textWidth > maxWidth) {
    finalFontSize = Math.floor(48 * (maxWidth / textWidth) * 0.95);
  }

  return (
    <div style={{ fontSize: finalFontSize, maxWidth, textAlign: 'center' }}>
      {text}
    </div>
  );
};
```

**关键点**：使用 `useTextSize` 钩子动态计算文本尺寸。

### 技巧 8: 数据驱动的图表动画

**问题**：图表数据难以动画化，经常出错

**解决方案**：

```typescript
import { BarChart } from 'recharts';

export const AnimatedBarChart = ({ data }) => {
  const frame = useCurrentFrame();
  const totalFrames = 120; // 4 秒 @ 30fps

  // 计算动画进度（0 到 1）
  const progress = Math.min(frame / totalFrames, 1);

  // 应用动画进度到数据
  const animatedData = data.map(item => ({
    ...item,
    value: item.value * progress
  }));

  return (
    <BarChart width={800} height={400} data={animatedData}>
      <Bar dataKey="value" fill={BRAND_COLORS.primary} />
    </BarChart>
  );
};
```

**关键点**：通过 `progress` 因子线性缩放所有值，实现整体动画。

```typescript
export const CountUpNumber = ({ from = 0, to, duration = 60 }) => {
  const frame = useCurrentFrame();
```

## 🎯 批量生成技巧

### 技巧 13: CSV 批量渲染最佳实践

**问题**：批量渲染经常失败或出错

**解决方案**：

```typescript
// scripts/batch-render.ts
import { renderMedia } from '@remotion/renderer';
import fs from 'fs';
import csv from 'csv-parser';

interface VideoJob {
  productName: string;
  price: number;
  imageUrl: string;
}

const renderBatch = async (csvFile: string) => {
  const jobs: VideoJob[] = [];

  // 读取 CSV
  fs.createReadStream(csvFile)
    .pipe(csv())
    .on('data', (row) => jobs.push(row))
    .on('end', async () => {
      // 限制并发数
      const CONCURRENCY = 2;
      const results = [];

      for (let i = 0; i < jobs.length; i += CONCURRENCY) {
        const batch = jobs.slice(i, i + CONCURRENCY);

        await Promise.all(batch.map(async (job) => {
          try {
            await renderMedia({
              composition: 'ProductVideo',
              serveUrl: 'http://localhost:3000',
              codec: 'h264',
              crf: 20,
              outputLocation: `./output/${job.productName}.mp4`,
              inputProps: job // 传递参数
            });
            console.log(`✅ 完成：${job.productName}`);
          } catch (error) {
            console.error(`❌ 失败：${job.productName}`, error);
            results.push({ job, status: 'failed', error });
          }
        }));
      }
    });
};

renderBatch('./products.csv');
```

**关键点**：限制并发数（通常 2-4），避免系统过载。

```typescript
    continue;
  }

  // 渲染...

  saveCompleted(job.productName);
}
```

**关键点**：保存已完成列表，支持从中断处恢复。

### 技巧 16: 音量标准化

```typescript
// 使用 ffmpeg-fluent 标准化音量
import ffmpeg from 'fluent-ffmpeg';

const normalizeAudio = (inputPath: string, outputPath: string) => {
  return new Promise((resolve, reject) => {
    ffmpeg(inputPath)
      .audioFilter('loudnorm=I=-16:TP=-1.5:LRA=11') // EBU R128 标准
      .output(outputPath)
      .on('end', resolve)
      .on('error', reject)
      .run();
  });
};

// 使用
```

## 🎁 快速参考表

### 常用值速查

| 参数 | 场景 | 推荐值 |
|------|------|-------|
| `fps` | 标准 | 30 |
| `fps` | 高帧率（平滑） | 60 |
| `crf` | 高质量 | 18-20 |
| `crf` | 平衡质量和大小 | 23-25 |
| `crf` | 小文件 | 28-30 |
| 并发数 | 开发中 | 1-2 |
| 并发数 | 生产 | CPU核数 / 2 |

### 性能基准

| 操作 | 耗时 | 优化建议 |
|------|------|--------|
| 30秒 1080p30fps | ~5秒 | ✅ 正常 |
| 2分钟 1080p30fps | ~30秒 | ✅ 可接受 |
| 5分钟 1080p30fps | ~1.5分钟 | ⚠️ 考虑缓存 |
| 10分钟+ | 很慢 | 🔴 使用 OffthreadVideo |
