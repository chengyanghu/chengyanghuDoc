## 🎯 通用技巧提示词

### 1. 创建参数化视频模板

```
帮我创建一个 Remotion 视频模板，支持通过参数动态控制：
- 标题文本
- 背景颜色（品牌色）

- 背景音乐

要求：
1. 使用 Zod 定义参数 schema
2. 支持在 Remotion Player 中预览时修改参数
3. 参数应该能在命令行渲染时指定
4. 代码应该易于复用

请提供完整的 Root.tsx 和 composition 代码。
```

### 2. 实现高性能文本动画

```
我需要在 Remotion 中实现一个高效的文本动画效果：
- 文本逐字出现
- 每个字符之间有 20ms 延迟
- 字符出现时带有 scale 和 opacity 动画
- 性能要足以处理 100+ 字符的长文本

要求：
1. 使用 useLayoutEffect 计算文本尺寸（避免 layout shift）
2. 使用 memo 优化每个字符组件
3. 避免不必要的重新渲染
4. 提供完整可用的 Hook 代码

提供可复用的 useCharacterAnimation Hook。
```

### 3. 音频驱动的视觉效果

```
我需要在 Remotion 中创建音频驱动的视觉效果：
- 根据音频频谱生成动态柱状图
- 低频率频段控制底部柱子高度
- 高频率频段控制顶部柱子高度
- 实时响应音乐节奏

要求：
1. 使用 AudioData 获取实时频谱数据
2. 使用 interpolate 将频谱映射到柱子高度
3. 添加平滑过渡（避免抖动）
4. 性能优化（使用 GPU 加速如果可能）

提供完整代码示例和最佳实践建议。
```

```

1. 内存管理：
   - 使用 OffthreadVideo 处理嵌入视频
   - 实现智能缓存（缓存渲染中间产物）
   - 及时释放不需要的资源
2. 渲染优化：
   - 使用渐进式渲染（分段渲染）
   - 调整并发度（根据 CPU 核心数）
   - 启用 GPU 加速（如果可用）
3. 代码优化：
   - 使用 useLayoutEffect 替代 useEffect
   - 使用 memo 包装复杂组件
   - 避免闭包陷阱和引用问题

提供完整的性能优化检查清单和代码示例。
```

### 提示词模板结构

最佳的 Remotion 提示词应包含：

```
1. 目标陈述：
   "我需要在 Remotion 中实现..."

2. 具体需求：
   - 功能 A
   - 功能 B
   - 功能 C

3. 约束条件：
   - 性能要求
   - 兼容性要求
   - 代码质量标准

4. 输出要求：
   - 代码示例
   - 最佳实践
   - 常见问题

5. 背景信息（如果有）：
   - 公司品牌色系
   - 现有组件库
   - 性能基准
```

### 高效 Prompt 技巧

1. **具体化**：使用数值而非模糊词汇
   - ❌ "快速的动画"
   - ✅ "500ms 的 spring 动画，tension 100"

2. **示例数据**：提供输入输出示例
   - 让 Claude 理解你想要的确切格式

3. **约束条件**：明确声明限制
   - 性能目标、浏览器兼容性、文件大小等

4. **拆分任务**：大任务分成小任务
   - 逐个提示而非一次性全部

5. **参考现有实践**：提到已知的好方案
   - "类似于 TikTok 的文本动画效果"

## 💡 最佳实践总结

### 五个黄金规则

1. **使用参数化** - 所有视频都应该支持参数配置
2. **重视性能** - 提前考虑内存和渲染速度
3. **组件化设计** - 创建可复用的动画和布局组件
4. **版本管理** - 跟踪模板版本和更新
5. **测试渲染** - 在全分辨率渲染前进行预检查

### 避免的常见错误

❌ 硬编码所有值 - 改用参数系统
❌ 忽视性能 - 使用 benchmark 工具
❌ 复杂的嵌套组件 - 扁平化结构
❌ 手动同步音频 - 使用时间计算
❌ 一次性渲染 - 使用缓存策略
